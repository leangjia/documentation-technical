Manifest

清单文件用于声明一个Python包作为Odoo模块，并指定模块的元数据。

It is a file called __openerp__.py and contains a single Python dictionary, where each key specifies module metadatum.

{
    'name': "A Module",
    'version': '1.0',
    'depends': ['base'],
    'author': "Author Name",
    'category': 'Category',
    'description': """
    Description text
    """,
    # data files always loaded at installation
    'data': [
        'mymodule_view.xml',
    ],
    # data files containing optionally loaded demonstration data
    'demo': [
        'demo_data.xml',
    ],
}

Available manifest fields are:

name (str, required)

模块的人类可读名称

version (str)

该模块的版本，应该遵循`语义versioning`_规则

description (str)

扩展描述为模块，在reStructuredText的

author (str)

的模块作者姓名

website (str)

网站的网址为模块作者

license (str, defaults: LGPL-3)

该模块分发许可证

category (str, default: Uncategorized)

在Odoo分类类别，该模块粗糙的业务字段。

虽然使用`现有categories`_建议，该字段为自由和未知类别的即时创建。可以使用分离器``创建类别层次结构/如``美孚/ Bar``将创建类``Foo，一类``Bar``为``Foo``子类，并将设置``Bar``作为模块的类别。

depends (list(str))

必须这个之前加载，或者是因为该模块使用他们所创造的功能或者是因为它改变了资源，他们定义Odoo模块。

当安装一个模块，其所有相关之前安装。同样地，一个模块被加载之前，依赖被加载。

data (list(str))

的必须始终安装或更新与模块数据文件列表。路径从模块根目录清单

demo (list(str))
List of data files which are only installed or updated in demonstration mode
auto_install (bool, default: False)

如果``True``，那么这个模块将被自动如果所有的依赖也都安装安装。

It is generally used for "link modules" implementing synergic integration between two otherwise independent modules.

例如``sale_crm``取决于双方``sale``和``crm``并设置为``auto_install``。当两个``sale``和``安装crm``，它会自动将客户关系管理的运动跟踪到销售订单没有任何``sa​​le``或``crm``意识到彼此
